template coding template to request API access to existing ESRI feature services using Esri's Developer OAuth 2.0

getting started

set up OAuth2.0 key

Sign in into the arcgis online account and select the "developers" application. Contact the department's IT/GIS team to provide an account with developer access.

create new application

create a new application, name and title should match the purpose

this will generate two main parts which will be used for API endpoint access

  • Client ID
  • Client Secret

    the third part to the endpoint access is the rest service address, which can be found by accessing the feature class address in content loaded to ARCGIS online. In this example we want to access the layer itself

running the script

depending on the ETL environment we'll need the following libraries

import pandas as pd
import requests

we'll bring the oauth tools back into the template as follows: 

# Custom Variables
## Feature Server must be of the actual spatial feature (point, line, area)
fsURL = "insert url here"

fieldsOut = "*"

## Arcgis Dashboard oAuth 2.0
client_id = "client id here"
client_secret = "client secret here"
tokenrequest = f"https://www.arcgis.com/sharing/rest/oauth2/token/?client_id={client_id}&client_secret={client_secret}&grant_type=client_credentials"

# Get Token
token_response = requests.get(tokenrequest)
token_json = token_response.json()
token = token_json['access_token']

# Get Features
features_url = f"{fsURL}/query?where=1%3D1&outFields={fieldsOut}&returnGeometry=true&f=json&token={token}"

features_response = requests.get(features_url)
features_json = features_response.json()

# Retrieve all features
features = features_json['features']
field_names = [field['name'] for field in features_json['fields']]
data = []


for feature in features:
    attributes = feature['attributes']
    data.append(attributes)

export_data = pd.DataFrame(data)

This should bring in the data as dataframes, and can be refreshed however many times. If the number of rows are more than 2000, use the following modification to work around the limit: 

# Get Features
batch_size = 2000  # Number of records per batch
offset = 0
features = []

while True:
    features_url = f"{fsURL}/query?where=1%3D1&outFields={fieldsOut}&returnGeometry=true&f=json&token={token}&resultOffset={offset}&resultRecordCount={batch_size}"
    features_response = requests.get(features_url)
    features_json = features_response.json()

    batch_features = features_json['features']
    features.extend(batch_features)

    if len(batch_features) < batch_size:
        break  # Reached the end of records
    offset += batch_size
